## 🎯 核心交互流程

### Step 1｜首页（极空）✅

```
--------------------------------
|                              |
|          给我一句            |
|     现在能站住的话            |
|                              |
|        [ 开始 ]               |
|                              |
--------------------------------
```

**设计要点：**
- ✅ 没有 logo
- ✅ 没有说明
- ✅ 极简设计
- ✅ 工具感，不是内容 App

### Step 2｜选择处境 ✅

```
你现在的状态是？
○ 我在犹豫
○ 我在退缩
○ 我在硬撑
○ 我在被动挨打
```

**设计要点：**
- ✅ 不是情绪（开心/难过）
- ✅ 是行动前状态
- ✅ 4 种处境分类

### Step 3｜语录呈现（冷处理）✅

```
"没有调查，
 就没有发言权。"

    ——《实践论》

   （向下滑动）
```

**设计要点：**
- ✅ 不解释
- ✅ 不高亮
- ✅ 不做金句海报
- ✅ 字体偏正文，不煽情
- ✅ "迎面撞上"，不是"被喂"

### Step 4｜行动确认（灵魂一刀）✅

```
面对这句话：
○ 我照做了
○ 我没照做
○ 我逃避了
```

**设计要点：**
- ✅ 不给"感受"
- ✅ 不给"是否有帮助"
- ✅ 只问：你站哪边

### Step 5｜结束（立刻退场）✅

```
已记录。
去做你的事。
```

**设计要点：**
- ✅ 不推荐下一句
- ✅ 不诱导继续使用
- ✅ 2 秒后自动退出

## 📊 数据结构

### 模型层（Models/）

**QuoteModels.swift:**
- `Situation` - 处境枚举（犹豫/退缩/硬撑/被动挨打）
- `Quote` - 语录模型（内容/出处/处境）
- `ActionChoice` - 行动选择（照做了/没照做/逃避了）
- `QuoteRecord` - 记录模型（日期/处境/语录/行动）

### 服务层（Services/）

**QuoteService.swift:**
- 管理 20+ 条毛选语录
- 按处境分类
- 随机抽取

**RecordService.swift:**
- 本地存储（UserDefaults）
- 记录管理
- 清空功能
