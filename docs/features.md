# 一、产品定位（一句话定生死）

**这是一个：
“当我犹豫、退缩、想逃的时候，用一句毛选逼我站直的工具。”**

不是学习，不是收藏，不是传播。

---

# 二、功能结构总览（极简，但有锋芒）

```
App
├── 抽一句（核心）
│   ├── 选择当前处境
│   ├── 抽取语录
│   └── 行动确认
│
├── 我的记录
│   ├── 我抽过的句子
│   ├── 我的选择（做 / 没做 / 逃避）
│   └── 时间线回看
│
├── 语录池（只读）
│   ├── 按“处境”分类
│   └── 原文 + 来源
│
└── 设置
    ├── 使用说明（很短）
    ├── 数据本地化
    └── 清空记录
```

**没有登录、没有云端、没有社交。**

---

# 三、核心交互 ①：抽一句（主路径）

这是整个 App **唯一被精心设计的路径**。

---

## 🟡 Step 1｜首页（极空）

```
--------------------------------
|                              |
|          给我一句            |
|     现在能站住的话            |
|                              |
|        [ 开始 ]               |
|                              |
--------------------------------
```

* 没有 logo
* 没有说明
* 用户一进来就知道：这是个**工具，不是内容 App**

---

## 🟡 Step 2｜选择「当前处境」

```
--------------------------------
|  你现在的状态是？             |
|                              |
|  ○ 我在犹豫                   |
|  ○ 我在退缩                   |
|  ○ 我在硬撑                   |
|  ○ 我在被动挨打               |
|                              |
|        [ 确认 ]               |
--------------------------------
```

**关键点：**

* 不是情绪（开心/难过）
* 是**行动前状态**
* 这是毛选真正工作的地方

---

## 🟡 Step 3｜语录呈现（冷处理）

```
--------------------------------
|                              |
|  “没有调查，                 |
|   就没有发言权。”            |
|                              |
|        ——《实践论》           |
|                              |
|   （向下滑动）                |
--------------------------------
```

**设计原则：**

* 不解释
* 不高亮
* 不做金句海报
* 字体偏正文，不煽情

> 这是“迎面撞上”，不是“被喂”。

---

## 🟡 Step 4｜行动确认（灵魂一刀）

```
--------------------------------
|  面对这句话：                 |
|                              |
|  ○ 我照做了                   |
|  ○ 我没照做                   |
|  ○ 我逃避了                   |
|                              |
|        [ 记录 ]               |
--------------------------------
```

* 不给“感受”
* 不给“是否有帮助”
* 只问：**你站哪边**

---

## 🟡 Step 5｜结束（立刻退场）

```
--------------------------------
|                              |
|     已记录。                  |
|                              |
|     去做你的事。              |
|                              |
--------------------------------
```

**不会推荐下一句，不会诱导继续使用。**

---

# 四、核心交互 ②：我的记录（残酷回看）

这是你**半年后会感谢自己的地方**。

---

## 时间线视图

```
--------------------------------
|  2025.03.18  犹豫             |
|  “没有调查，就没有发言权”    |
|   → 我逃避了                  |
|                              |
|  2025.02.04  硬撑             |
|  “革命不是请客吃饭”          |
|   → 我照做了                  |
--------------------------------
```

* 不做统计
* 不给总结
* 只是**事实陈列**

---

## 点击某条记录

```
--------------------------------
|  状态：犹豫                   |
|  时间：2025.03.18             |
|                              |
|  “没有调查，                 |
|   就没有发言权。”            |
|                              |
|  我的选择：逃避               |
|                              |
--------------------------------
```

**没有“后悔按钮”。**

---

# 五、语录池设计（为你兜底）

这是防止用户觉得“是不是就这几句”的地方。

```
语录池
├── 犹豫时该看的
│   ├── 实践 / 调查
│   └── 判断与发言
│
├── 退缩时该看的
│   ├── 斗争 / 代价
│   └── 主动性
│
├── 硬撑时该看的
│   ├── 长期战
│   └── 意志
│
└── 被动挨打时该看的
    ├── 敌我关系
    └── 现实判断
```

**只读，不可收藏，不可点赞。**

---

# 六、你现在就能用的「交互流程图（文字版）」

```
启动 App
   ↓
首页「给我一句」
   ↓
选择处境
   ↓
随机抽取（按处境）
   ↓
展示语录
   ↓
行动确认
   ↓
记录入库
   ↓
退出 / 回首页
```

这是一个**天然防沉迷结构**。

---

# 七、我给你的产品底线（请你认真看）

* 这个 App **不该让人感觉舒服**
* 它应该像一个**沉默、但站得很直的人**
* 它存在的意义，是在你想逃的时候，
  **不给你台阶**
